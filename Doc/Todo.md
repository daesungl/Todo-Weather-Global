# (완료) 필요한 작업. 잘못되었다고 생각들거나 더 좋은 의견이 있다면 언제든지 이야기해줘.
1. 스텝을 수정하고 저장하는 모달에서 저장 버튼 누르면 시간이 지체되는데 이때 로딩 인디케이터를 보여주고 있어. 그런데 저장 버튼과 모달 중간에 2개 로딩 인디케이터가 뜨는데 display name 편집 체크 버튼처럼 저장 버튼 누르면 저장 버튼 안에만 로딩 인디케이터를 표시하면 될 것 같아.
2. 플랜 > 스텝을 편집하면 로컬에는 저장이 되는데 재접속하면 저장이 안되어있어. 파이어베이스에 저장이 되도록 해야할 것 같아. 
3. 접속하면 정보 불러오는게 느린데 로컬에 저장본이 있으면 로컬 저장본을 먼저 보여주고 데이터베이스에서 불러온 것들로 업데이트되도록 해야 할 것 같아. 지금은 로딩이 꽤 체감되는 수준.  
4. 플랜 > 상세 스텝 페이지에서 페이지 전체를 우측으로 드래그하면 뒤로가기가 되는건 너무 좋은데 가끔 페이지 전체가 어정쩡하게 멈춰서 페이지가 화면 우측 끝에 붙어서 레이아웃이 틀어지는 경우가 있어. 안정감 있는 스와이프, 제스쳐 동작이 되도록 개선할 부분이 있는지 확인 필요.
5. 플랜 > 스텝의 알림은 owner 에게만 가는건가? 이 알림 설정은 그 플랜에 참여하고 있는 멤버의 권한과 무관하게 각각 설정할 수 있게 해야 할 것 같아.
5.1 플랜 > 스텝의 알림은 작성자, 편집자, 뷰어 모두가 각각 설정한 대로 알람이 울려야 함.
5.2 플랜 > 신규 스텝 생성 시, 알람 활성화를 했다면 그 알람 설정은 작성자에게만 해당됨. 공유된 스텝의 기본은 알람이 끈 상태로 세팅됨. 이후 각자 알람을 켜거나 끄거나 했다면 그 유저에게는 본인이 설정한 알람으로 알람 설정 반영.
5.3 플랜 > 신규 스텝 생성이 아니라 편집에서 알람을 수정한 경우에도 편집자에게만 알람이 반영됨.
