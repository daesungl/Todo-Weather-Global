1. WeatherAPI 공식 Explorer 활용 (가장 확실함)
WeatherAPI 웹사이트에서 제공하는 API Explorer를 사용하면 편리합니다.

우리가 코드에 넣은 lat, lon 값을 그대로 입력하고 결과를 조회해 보세요.
앱에 나오는 데이터와 Explorer의 Raw JSON 결과가 일치하는지 보면 "데이터가 중간에 왜곡되지 않고 잘 전달되는지"를 100% 확인할 수 있습니다.
2. 구글(Google) 날씨와 비교 🔍
구글 검색창에 Weather in [지역명] (예: Weather in Jeju)을 검색하면 나오는 날씨 카드는 주로 Weather.com이나 글로벌 기상 관측소 데이터를 사용합니다.

**자외선(UV)**과 가시거리 항목도 구글 날씨 상세 정보에 포함되어 있습니다.
글로벌 서비스들끼리는 데이터 소스가 비슷하기 때문에 구글 날씨와 수치가 비슷하다면 데이터의 신뢰도가 높다고 판단할 수 있습니다.
3. 터미널 로그 실시간 모니터링 (우리가 심어둔 것!)
아까 제가 로그를 심어두었죠? 터미널에서 앱을 조작할 때 찍히는 로그를 보세요.

[GlobalService] Extra data received: UV=3, Vis=15

이 로그의 값과 앱 화면의 값이 일치하는지만 봐도 "연동 오류" 여부는 바로 검증됩니다.
4. 대표적인 해외 도시로 테스트 ✈️
우리나라는 기상청 데이터가 우선이라 섞여서 보일 수 있지만, **런던(London)**이나 뉴욕(New York) 같은 해외 도시를 즐겨찾기에 추가해서 들어가 보세요.

해외 지역은 모든 데이터를 WeatherAPI에서 100% 가져오도록 되어 있습니다.
해외 지역의 예보와 가시거리가 상식적인 범위 내에서 잘 나오는지 보면 엔진 전체의 완성도를 체크하기 좋습니다.
꿀팁 💡: 가시거리가 15km 이상이면 매우 맑음, 2km 미만이면 안개/운무가 심한 상태라고 보시면 데이터의 정합성을 판단하기 쉬워집니다! 🚀😊
