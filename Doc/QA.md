1. WeatherAPI 공식 Explorer 활용 (가장 확실함)
WeatherAPI 웹사이트에서 제공하는 API Explorer를 사용하면 편리합니다.

우리가 코드에 넣은 lat, lon 값을 그대로 입력하고 결과를 조회해 보세요.
앱에 나오는 데이터와 Explorer의 Raw JSON 결과가 일치하는지 보면 "데이터가 중간에 왜곡되지 않고 잘 전달되는지"를 100% 확인할 수 있습니다.
2. 구글(Google) 날씨와 비교 🔍
구글 검색창에 Weather in [지역명] (예: Weather in Jeju)을 검색하면 나오는 날씨 카드는 주로 Weather.com이나 글로벌 기상 관측소 데이터를 사용합니다.

**자외선(UV)**과 가시거리 항목도 구글 날씨 상세 정보에 포함되어 있습니다.
글로벌 서비스들끼리는 데이터 소스가 비슷하기 때문에 구글 날씨와 수치가 비슷하다면 데이터의 신뢰도가 높다고 판단할 수 있습니다.
3. 터미널 로그 실시간 모니터링 (우리가 심어둔 것!)
아까 제가 로그를 심어두었죠? 터미널에서 앱을 조작할 때 찍히는 로그를 보세요.

[GlobalService] Extra data received: UV=3, Vis=15

이 로그의 값과 앱 화면의 값이 일치하는지만 봐도 "연동 오류" 여부는 바로 검증됩니다.
4. 대표적인 해외 도시로 테스트 ✈️
우리나라는 기상청 데이터가 우선이라 섞여서 보일 수 있지만, **런던(London)**이나 뉴욕(New York) 같은 해외 도시를 즐겨찾기에 추가해서 들어가 보세요.

해외 지역은 모든 데이터를 WeatherAPI에서 100% 가져오도록 되어 있습니다.
해외 지역의 예보와 가시거리가 상식적인 범위 내에서 잘 나오는지 보면 엔진 전체의 완성도를 체크하기 좋습니다.
꿀팁 💡: 가시거리가 15km 이상이면 매우 맑음, 2km 미만이면 안개/운무가 심한 상태라고 보시면 데이터의 정합성을 판단하기 쉬워집니다! 🚀😊

 WeatherAPI의 현실적인 한계                                           
                                                                       
  WeatherAPI는 글로벌 모델(GFS 기반) 데이터를 가공해서 제공합니다.     
  소비자용 앱에는 충분하지만, 정밀도 면에서 한계가 있습니다:           
  - 격자 해상도가 낮아 산간 지역이나 해안가 오차가 큼                  
  - 단기 강수 예보 정확도가 낮음                                       
  - 기본적으로 "중간 수준의 글로벌 평균치"
                                                                       
  진지하게 고려할 대안: Open-Meteo                                     
                                                                       
  검색 결과에서 가장 눈에 띄는 대안입니다.                             
                                                                       
  ┌────────────┬───────────────────┬───────────────────────────────┐   
  │            │    WeatherAPI     │          Open-Meteo           │
  ├────────────┼───────────────────┼───────────────────────────────┤   
  │ 데이터     │ GFS 등 글로벌     │ ECMWF + 각국 기상청 (최고     │
  │ 소스       │ 모델              │ 등급)                         │
  ├────────────┼───────────────────┼───────────────────────────────┤   
  │ 해상도     │ ~11km             │ 유럽/미국 1~2km               │
  ├────────────┼───────────────────┼───────────────────────────────┤   
  │ 비용       │ 현재 사용 중      │ 비상업: 무료 / 상업: 구독제   │
  ├────────────┼───────────────────┼───────────────────────────────┤   
  │ 장소 검색  │ 포함              │ 없음 (별도 필요)              │
  └────────────┴───────────────────┴───────────────────────────────┘   
                  
  ECMWF는 기상청 수준의 모델로, WeatherAPI보다 품질이 확연히 높습니다. 
                  
  현실적인 판단                                                        
                  
  다만 한 가지 중요한 포인트가 있습니다. 현재 앱에서 WeatherAPI가 날씨 
  데이터 + 장소 검색을 동시에 담당하고 있습니다. Open-Meteo로 날씨를
  교체하면 장소 검색은 WeatherAPI를 유지하거나 별도 솔루션(Nominatim   
  등)이 필요합니다.

  제 생각엔 글로벌 출시를 진지하게 본다면 Open-Meteo로 교체할 가치는   
  충분합니다. 단, 지금 당장보다는 MVP 반응을 보고 결정해도 늦지
  않습니다. WeatherAPI가 "못 쓸 수준"은 아니고, 앱 초기 성패는 데이터  
  정밀도보다 UX가 먼저 결정하거든요.

⏺ 네, 광고 붙이는 건 상업 이용에 해당됩니다. Open-Meteo 약관에 명시되어
   있습니다:                                                           
                                                                       
  ▎ "앱이나 웹사이트에 광고를 표시하거나 구독을 운영하는 것은 상업적   
  ▎ 이용으로 간주된다"                                                 
                                                                       
  따라서 광고 수익 앱에 무료 API를 쓰면 약관 위반입니다. 유료 플랜으로 
  가야 합니다.                                                         
                                                                       
  Open-Meteo 상업용 플랜 가격이 공개 페이지에 명확히 안 나와 있어서    
  정확한 금액은 직접 확인이 필요한데, 구조는 "호출 수 기반이 아닌 고정 
  구독제" 라 사용자가 늘어도 예측 가능한 비용이 드는 건 장점입니다.    
                                                                       
  결론적으로 광고 앱 기준으로 옵션 정리하면:                           
  
  ┌──────────────┬──────────────┬─────────────────┐                    
  │              │  WeatherAPI  │ Open-Meteo 상업 │
  ├──────────────┼──────────────┼─────────────────┤                    
  │ 광고 앱 허용 │ 허용         │ 유료 플랜 필요  │
  ├──────────────┼──────────────┼─────────────────┤
  │ 현재 비용    │ 이미 사용 중 │ 별도 구독 추가  │
  ├──────────────┼──────────────┼─────────────────┤                    
  │ 데이터 품질  │ 중간         │ 상위            │
  ├──────────────┼──────────────┼─────────────────┤                    
  │ 장소 검색    │ 포함         │ 별도 필요       │
  └──────────────┴──────────────┴─────────────────┘                    
                  
  MVP 단계에서 광고를 붙일 계획이라면, WeatherAPI 유지하다가 사용자    
  반응 보고 Open-Meteo 상업 플랜으로 업그레이드하는 게 현실적인 순서일
  것 같습니다.
